# GitHub notes

## Caso normal: subir cambios a ExtFW

Sirve para cualquier cambio del proyecto:

```bash
git add .
git commit -m "Describe el cambio"
git push origin main
```

Esto sube los cambios al repo principal `ExtFW`.

## Noxtr

`noxtr` tiene su repo propio: `ExtFW-module-Noxtr`.
En tu ordenador puede seguir estando en `_modules_/noxtr`.

Esto solo vale despues de haber hecho la conversion inicial de `_modules_/noxtr` a repo propio.

Para subir cambios de `noxtr` a su repo:

```bash
cd _modules_/noxtr
git add .
git commit -m "Describe el cambio"
git push origin main
```

## Bajar cambios de GitHub a este ordenador

```bash
git pull origin main
```

Para bajar cambios de `noxtr`:

```bash
cd _modules_/noxtr
git pull origin main
```

## Deshacer cambios locales de un archivo

```bash
git restore ruta/archivo
```

Ejemplo:

```bash
git restore _includes_/autoloader.php
```

Esto deja ese archivo como estaba en el ultimo commit de Git.

## Quitar una carpeta del repo ExtFW pero dejarla en disco

```bash
git rm -r --cached _modules_/noxtr
printf "/_modules_/noxtr\n" >> .git/info/exclude
git commit -m "Remove noxtr module from ExtFW repository"
git push origin main
```

## Conversion inicial: hacer que `_modules_/noxtr` pase a ser su repo propio

Esto se hace una sola vez.

```bash
mv _modules_/noxtr /tmp/noxtr-backup
git clone https://github.com/Nailuj2k/ExtFW-module-Noxtr.git _modules_/noxtr
```

## Resumen

- `ExtFW` es el repo principal.
- `noxtr` se mantiene en `ExtFW-module-Noxtr`.
- Si quieres subir a `ExtFW`, basta con `git add .`, `git commit`, `git push`.
- Si quieres subir `noxtr`, entra en `_modules_/noxtr` y haz su `git add`, `git commit`, `git push`.
